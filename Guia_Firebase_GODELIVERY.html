Passo a Passo: Migração para Firebase

Angular 19 + Firebase Authentication + Firestore

Guia completo para configurar e rodar o app com backend real no Firebase

1. Introdução

Este guia explica como configurar o projeto GODELIVERY (versão Angular) para utilizar o Firebase como backend. A migração já foi aplicada no código-fonte. Agora você só precisa criar o projeto no Firebase Console e inserir as credenciais.

O que foi migrado:
  • Autenticação real (login, cadastro, logout, reset de senha via Firebase Auth)
  • Dados de restaurantes e produtos vindos do Firestore
  • Favoritos salvos por usuário (subcoleção)
  • Pedidos criados e listados a partir do Firestore
  • Seed automático de dados na primeira execução da Home

2. Pré-requisitos

  • Node.js 18 ou superior instalado
  • Conta Google (para acessar o Firebase Console)
  • O projeto já deve estar na pasta: C:\Users\luiz_kuhn\Downloads\GODELIVERY (1)
  • Dependências já instaladas (npm install já foi executado)

3. Criar e Configurar o Projeto no Firebase

3.1 Acessar o Console

  1. Abra o navegador e acesse: https://console.firebase.google.com/
  2. Faça login com sua conta Google.

3.2 Criar novo projeto

  1. Clique em "Criar um projeto" (ou "Add project").
  2. Dê um nome (ex: godelivery-seunome).
  3. Aceite os termos e clique em Continuar.
  4. Desative o Google Analytics por enquanto (pode ativar depois).
  5. Clique em "Criar projeto" e aguarde (≈ 1 minuto).

3.3 Ativar Authentication (Email/Senha)

  1. No menu lateral esquerdo, clique em Authentication.
  2. Clique em "Começar" / "Get started".
  3. Vá na aba Sign-in method.
  4. Clique em Email/Password → Ative a opção e clique em Salvar.

3.4 Ativar Firestore Database

  1. No menu lateral, clique em Firestore Database.
  2. Clique em "Criar banco de dados".
  3. Escolha "Iniciar em modo de teste" (para desenvolvimento).
  4. Selecione uma localização próxima (ex: southamerica-east1 - São Paulo).
  5. Clique em Ativar.
Atenção: O modo de teste expira em 30 dias. Depois configure regras de segurança (veja seção 7).

3.5 Obter as credenciais do app Web

  1. No canto superior direito (engrenagem ao lado de "Visão geral do projeto"), clique em Configurações do projeto.
  2. Role até "Seus apps" e clique no ícone da Web (</>).
  3. Dê um apelido (ex: GODELIVERY Web) e clique em Registrar app.
  4. Copie todo o objeto firebaseConfig que aparece na tela.
  5. Clique em Continuar para o console.

4. Inserir as Credenciais no Projeto Angular

4.1 Editar o arquivo de ambiente

Abra o arquivo com VS Code ou seu editor preferido:

C:\Users\luiz_kuhn\Downloads\GODELIVERY (1)\src\environments\environment.ts

Substitua o conteúdo pelo exemplo abaixo, colando os valores reais do Firebase:

export const environment = {
  production: false,
  firebase: {
    apiKey: "AIzaSyA...sua-chave-real...",
    authDomain: "seu-projeto-id.firebaseapp.com",
    projectId: "seu-projeto-id",
    storageBucket: "seu-projeto-id.appspot.com",
    messagingSenderId: "123456789012",
    appId: "1:123456789012:web:abc123def4567890"
  }
};

Repita o mesmo no arquivo environment.prod.ts.

6. Testando os Fluxos Principais

  • Cadastro: /signup → cria usuário no Auth + perfil no Firestore.
  • Login: usa as credenciais reais do Firebase.
  • Home: restaurantes carregados do Firestore + favoritos por usuário.
  • Restaurante / Produto: dados dinâmicos pelo ID.
  • Checkout: salva o pedido na coleção orders.
  • Meus Pedidos: lista os pedidos reais do usuário logado.
  • Perfil + Logout: dados do Firestore + logout real.

7. Regras de Segurança do Firestore (Muito Importante)

Enquanto estiver em modo de teste, qualquer pessoa pode acessar seus dados. Configure regras reais:

  1. No Firebase Console → Firestore Database → aba Regras.
  2. Substitua por este conteúdo e clique em Publicar:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /users/{userId}/favorites/{docId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /orders/{orderId} {
      allow create: if request.auth != null;
      allow read: if request.auth != null && resource.data.userId == request.auth.uid;
    }
    match /restaurants/{docId} { allow read: if true; }
    match /products/{docId} { allow read: if true; }
  }
}

8. Problemas Comuns

auth/invalid-api-key ou credenciais erradas
Verifique se colou as chaves corretamente no environment.ts. O projectId deve ser idêntico ao do console.
permission-denied
Verifique se você está logado e se as regras de segurança permitem a operação. Abra o DevTools (F12) → Console para ver a mensagem detalhada.
Dados não aparecem
Abra o Firestore no console. Se as coleções estiverem vazias, delete-as e recarregue a Home para rodar o seed novamente.

9. Deploy no Firebase Hosting (Opcional - Bônus)

  1. Instale o Firebase CLI (uma vez):
    npm install -g firebase-tools
  2. Faça login: firebase login
  3. Inicialize: firebase init hosting (escolha seu projeto, public dir = dist/godelivery, SPA rewrites = yes)
  4. Build: npm run build
  5. Deploy: firebase deploy --only hosting

Seu app ficará disponível em: https://seu-projeto-id.web.app
